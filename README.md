# Self-OS Optimizer (SOO) - システム要件定義書

## 1. 概要（エグゼクティブ・サマリ）
本システム「SOO」は、極めて高い「行動力（Do）」を持つ単一ユーザーに向けた、目的達成特化型の自己管理アプリケーションである。
WBS（目的定義）、自己資源（HP/MP）、高度なプロセス管理（PERT/フローチャート）、およびLLM（Gemini API）による予実分析・自動計画を統合。ユーザーを「意思決定」と「管理」の負荷から完全に解放し、持続可能なトップパフォーマンスの維持を実現する。

---

## 2. システム・アーキテクチャ（ゼロコスト・サーバーレス構成）
ランニングコストを月額0円（※Gemini API課金を除く）に抑えるため、PaaS/BaaSの無料枠（Free Tier）を最大限活用するモダンなサーバーレスアーキテクチャを採用する。

* **Frontend & Backend**: React (Next.js) 
  * インフラ: **Vercel** または **Cloudflare Pages** (ホスティング無料枠)
  * バックエンド処理は別サーバーを立てず、Next.jsの API Routes (Serverless Functions) に集約。
* **Database & Auth**: **Supabase** (PostgreSQL) または **Firebase** (Firestore/Auth)
  * シングルユーザーのため、無料枠の範囲内で十分に運用可能。
* **AI Engine**: Gemini 1.5 Pro / Flash API
* **External API**: Google Calendar API, TimeTree API

---

## 3. 機能要件 (Functional Requirements)

### 3.1 目的の構造化管理（Vertical Strategy）
* **WBS展開機能**: 「KGI（最終目的）」を入力すると、GeminiがKPI、マイルストーン、末端のToDoまで最大5階層のWBS構造を自動構築する。
* **進捗・価値のトレーサビリティ**: 末端タスクの完了が上位目的の進捗率（%）に自動反映される。また、完了時に「上位目的に与えた付加価値」を定性メモとして記録・蓄積する。

### 3.2 資源の動的管理とセーフティ（Dynamic Resource）
* **カスタム資源定義**: HP（体力）、MP（精神力・集中力）、時間、予算など、管理対象の資源を任意に定義・拡張可能。
* **リアルタイム消費ロジック**: タスク実行中、設定された「消費レート」に基づき資源がリアルタイムに減算される。
* **強制セーフティ・リスケジュール**: 
  * HP/MPが規定の閾値（例: 0）に達した場合、実行中タスクに強制ストップをかける。
  * 未完了タスクを、外部カレンダーの空き枠を計算した上で「翌日以降」へ自動でリスケジュールする。

### 3.3 ナレッジ主導のプロセス管理（Tactical Process & Knowledge）
* **外部カレンダー同期**: Google Calendar および TimeTree から「動かせない予定（FIX予定）」を双方向で同期する。
* **高度なプロセス・ビュー（マネジメントナレッジ適用）**:
  
  * **フローチャート/PERT図**: タスク間の依存関係（順序）を視覚化し、クリティカルパス（遅延が全体に影響する経路）をAIが自動特定してハイライトする。
  * **戦略的バッファの配置**: 予実のズレを吸収するため、スケジュール内に明示的なバッファ時間を設ける。このバッファ時間は原則として**「何もしない時間（完全な休息・空白）」**としてシステム上で保護され、安易にタスクで埋められない仕様とする。
* **ハイブリッド変換**: フローチャート（論理）からガントチャート（時間）、そしてToDoリスト（実行）へシームレスにデータが連動する。

### 3.4 予実管理と実行記録（Plan vs Actual）
* **タイムライン予実比較**:
  
  * 朝のAIリコメンド計画（Plan）と、実際の行動ログ（Actual）を上下に並べてガントチャート形式で可視化する。
  * 「タスクの開始・終了時刻」のズレを分単位で記録・比較する。
* **リソース予実比較**: 「見積もっていた消費HP/MP」と「実際に消費したHP/MP」の差分をグラフ化し、自己キャパシティの認識のズレを可視化する。
* **フォーカスモード（実行UI）**: 実行中は「現在取り組むべき1つのタスク」と「消費中の資源」のみを表示し、他の情報を隠蔽する。

### 3.5 AI駆動の自動改善ループ（Intelligence Learning）
* **07:00 自動計画リコメンド**:
  * 毎朝7:00、外部予定、現在のHP/MP残量、およびタスクの依存関係を計算し、Geminiが「今日の最適ルート」を生成・提示する。
* **自動ふりかえりレビュー**:
  * 一日の終わりに、タスク実行ログ（予実差）、資源消費、定性メモをGeminiへ送信。
  * 予実の乖離原因（見積もりの甘さか、資源の枯渇か等）を分析し、翌日への改善レポートを生成する。
* **コスト自動学習**: 予測コストと実績コストの乖離データを蓄積し、次回の同種タスクの「デフォルト消費HP/MP」をシステムが自動で補正・最適化する。

## 4. 非機能要件 (Non-Functional Requirements)

* **ランニングコスト**: 0円（Gemini APIの実費を除く）。Vercel、Supabase等のFree Tier枠内に収まるよう、APIのポーリング頻度やDBへの書き込み回数を最適化する。
* **ユーザー想定**: 1名（自分専用。徹底的なパーソナライズを前提とする）。
* **クロスプラットフォーム**: PCブラウザおよびAndroidスマートフォン（PWA）での利用。
* **UX/UI制約 (楽観的UIの実装)**: モバイルでの操作摩擦を極限まで減らすため、1タップ操作を主軸とする。また、サーバーレス特有のコールドスタートによるUIのフリーズを防ぐため、通信完了を待たずに画面上のステータスを更新する「楽観的UI（Optimistic UI）」を実装する。
